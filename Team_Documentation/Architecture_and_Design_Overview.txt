Osmosis: Project Architecture & Design Overview

1. Project Vision
Osmosis is an interactive, scholarly vocabulary application designed to elevate the user's lexicon through an immersive, multi-round learning experience. The goal was to build a robust, scalable app without the overhead of heavy conceptual frameworks like React or Angular, ensuring maximum performance, rapid load times, and easy maintainability.

2. Design & Aesthetics
Aesthetic evolution: We transitioned from a traditional, heavy "scholarly brown" palette to a vibrant, modern "Intellectual Green & Gold" theme. 
- Primary Color: "Duolingo Green" variants to provide modern familiarity and keep users engaged with bright, psychologically rewarding tones.
- Accent Color: "Intellectual Gold" replacing the flat borders with a rich, premium aesthetic.
This combination elevates the UI, making learning feel both like a prestigious academic endeavor and an addicting gamified experience.

3. Architecture Details
- Core Structure: The app follows a Multi-Page Application (MPA) architecture. Each conceptual round (Round 1, Round 2, etc.) runs on its own isolated HTML file. This enforces strict separation of concerns while keeping bundle sizes near zero since there's no complex routing JS.
- State Management: We utilize the Browser's native `sessionStorage` and `localStorage` to persist state across page navigation. `sharedState` in `shared.js` manages the user's score, username, and active words effortlessly across the ecosystem.
- API Integrations: The app dynamically interfaces with the Datamuse API (to fetch relevant scientific/academic words) and DictionaryAPI (to fetch definitions). It includes an embedded fallback JSON dictionary in case of network failures, ensuring offline/low-bandwidth resilience.
- Sound Design: Real-time, zero-dependency audio synthesis. Instead of relying on heavy `.mp3` files, `AudioManager` uses the native Web Audio API (`AudioContext`) to dynamically synthesize clicks, chips, and success chords using sine and sawtooth oscillators.

4. UI Implementations
- Modular "Stitch UI": We seamlessly integrated a custom UI template. The CSS uses modern variables (`--parchment`, `--ink`, etc.), linear/radial gradients, and flex/grid systems for pure responsive formatting.
- Dynamic Authentication: A robust, cookie-free login system. The app remembers previously entered user identities and elegantly adapts the DOM to display a personalized "Welcome back, {Name}" layout without any backend database or heavy authentication providers.

5. Multiplayer Infrastructure (WebSockets)
- The app supports a synchronized intellectual duel via real-time WebSockets (`Socket.IO`).
- A lightweight Node server (`server.js`) handles room allocations.
- Dynamic Injection: Scripts are injected only when a user possesses a valid `mp_room` key in their session, keeping single-player modes completely isolated and fast.
